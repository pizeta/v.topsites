NAME

v.topsites - Topsites detection using segmentation and filtering

KEYWORDS

vector, DTM filtering

SYNOPSIS

v.topsites
v.topsites help
v.topsites [-k] dtm=name [resolution=float] [alpha=float] [threshold_comp=float] [threshold_curv=float] [--verbose] [--quiet]

Flags:

-k
Keep (do not delete) intermediate maps
--verbose
Verbose module output
--quiet
Quiet module output

Parameters:

dtm=name
Name of input raster map representing a Digital Terrain Model
resolution=float
Resolution used for the analysis. If a value is not specified, the resolution of the current region is used.
alpha=float
Value of the alpha parameter used for the DTM segmentation, default is 25
threshold_comp=float
Compactness threshold for areas filtering, default is 2
threshold_curv=float
Curvature threshold for areas filtering, default is 0.001

DESCRIPTION

v.topsites automatically detects hill-top sites candidates from a DTM.
This is accomplished in two steps:
  • the DTM is segmented using GRASS' r.seg module (http://www.ing.unitn.it/~vittia/sw/)
  • the resulting areas are filtered according to compactness and mean curvature of the terrain in each area
The module creates one map containing the hill-top sites candidate areas and nine intermediate maps.
The output map is a vector map named as the input DTM map plus the "_hilltopsites" suffix. For example, if the input DTM map is named "elevation", the output hill-top sites map is named "elevation_hilltopsites".

Nine intermediate maps are created and automatically deleted at the end of the process, unless the k flag is selected. If $dtm identifies the DTM input map name:

  • $dtm"_prof_curv" is the name of the profile curvature raster map
  • $dtm"_seg" is the name of the segmented DTM raster map
  • $dtm"_disc" is the name of the DTM discontinuities raster map
  • $dtm"_clump" is the name of the DTM discontinuities raster map, after clumping areas
  • $dtm"_clump" is the name of the DTM discontinuities vector map, after clumping areas
  • $dtm"_aver_curv" is the name of raster map containing the average curvature for each area
  • $dtm"_points" is the name of the vector map containing clumped areas, with centroids transformed to points
  • $dtm"_areas" is the name of the vector map containing clumped areas, with centroids transformed to points
  • $dtm"_points" is the name of the vector map containing clumped areas, with points transformed back to centroids, with a complete attribute table, as specified below. Areas are not filtered by compactness and curvature
A brief description of this approach and an introductory discussion of the results can be found in "Elisa Rosciano, Fabio Cavulli, Marco Ciolli, Alfonso Vitti, Paolo Zatelli, DTM segmentation and analysis for archaeological hill-top sites recognition in GRASS, 2011, Geomatics Workbooks", available here http://geomatica.como.polimi.it/workbooks/n10/GW10-FOSS4Git_2011.pdf. Further papers are in preparation.

EXAMPLE

Create the "elevation_hilltopsite" map with resolution 10 map units (usually meters), segmenting the DTM map with value of the alpha parameter equal to 22, filtering the resulting areas by selecting those whose compactness is lower than 2.2 and mean curvature is higher than 0.002. 
v.topsites dtm=elevation resolution=10 alpha=22 threshold_comp=2.2 threshold_curv=0.002
Create the "dtm001_hilltopsite" map with resolution of the current region, segmenting the DTM map with value of the alpha parameter equal to 25, filtering the resulting areas by selecting those whose compactness is lower than 2 and mean curvature is higher than 0.001. 
v.topsites dtm=dtm001
